Software Tester

A software tester is involved in the quality assurance phase of software development and deployment. They are required to perform automated and manual testing to ensure that the software created by developers is fit for purpose and remove any bugs or issues in the product before it is deployed to everyday users.

Their roles are integral to creating software systems and technology products, including vehicles, electronics, defence, and healthcare. They may work on bespoke, stand-alone or multinational projects globally, costing billions of pounds. Thus they are required to familiarise themselves with multiple programming and coding languages.

These are some of the responsibilities for software testers:

  • Monitoring applications and software systems
  • Conduct stress testing, performance testing, functional testing and scalability testing
  • Write and execute test scripts
  • Run manual and automated tests
  • Provide objective feedback to software development project teams
  • Troubleshooting and Problem Resolution
  • File analysis
  • Liaise with project teams in other parts of the world
  • Communicate findings to technical and non-technical colleagues
  • Meet with system users to understand project scope
  • Collaborate with software developers and project support teams

Work Scopes

Software testers’ workplaces are mostly office-based and they will spend most of their time in front of a computer. Many large graduate employers offer summer internships and one-year industry placements, which provide the opportunity to gain relevant work experience. Completing such an internship will provide useful professional software testing experience and let people understand the complete development lifecycle. Besides that, some companies run job shadowing programs to gain insight into the software testing industry. However, some companies will allow software testers to work from home allowing more flexibility working time between the company and the software tester.

More importantly, a software tester’s job is to report their findings on bugs and errors of the code to the software developer, manager in charge of the project and company. They are not allowed to make any changes to the code even if it’s just minor bugs or mistakes because it is not just a matter of experience or skill. Code is highly customised and interdependent. Even experienced developers avoid stepping on each other's code until they understand all the dependencies involved. Moreover, not every bug is a bug. Sometimes extra code is intentionally added to support some future functionality or to act as a temporary placeholder.

Finally, quick fixes are not always properly documented. And you end up breaking the version history, reducing the likelihood that everyone is working on the latest iteration of the product. This is why the best approach for software testers is to separate testing and development responsibilities or they will put the project at risk. With that said, software testers are not very commonly known because their job is to ensure the code has no bugs or errors that will cause danger to both customer and company. Thus, to become software testers we will need to have the determination to become unsung heroes.

Web Developer

A Web Developer is responsible for programming code that tells websites and web applications how to operate. Web Developers typically specialize in either “front-end” (“client-side”) development or “back-end” (“server-side”) development. Some versatile and highly-sought-after professionals do both, and they’re called “Full-Stack Developers.”

Web Developers work on implementing visual designs and layouts, integrating graphics and content, optimizing website performance and capacity, and testing to ensure solid performance across any type of computer, mobile or device screen. No matter your exact job, if you want to become a Web Developer you would be expected to understand how to write code in some widely used web programming languages including JavaScript, HTML (HyperText Markup Language), CSS (Cascading Style Sheets), and more.

It’s also likely that any Web Developer would spend a lot of time collaborating and communicating with others on larger development teams, including Software Developers, Web Designers, Project Managers, and other stakeholders.

Web Developers typically work either as independent contractors/freelancers or as a traditional employees. As freelancers, web developers collaborate with clients individually. As full-time employees, web developers typically work on a team, regularly interacting with other developers, web designers, IT professionals, and management.

The main duties of web developers include:

  1. Meeting with clients/management:
    Web developers communicate with clients or management to see their vision and ideas about the website the developer would be working on.
  2. Write the code:
    Web developers spend most of their time writing code for websites in languages like HTML, CSS, and Java. Depending on their specialization, web developers may focus on front-end code, back-end code, or both.
  3. Testing:
    In addition to building a site's structure, web developers must ensure the site functions accurately and responsively. Web developers create custom controls, dynamic pages, navigation menus, and mobile interfaces. Developers test elements of the user interface to identify and fix bugs.
  4. Coordinate with Other Team Members:
    Web developers often coordinate with other professionals to bring a website design to fruition on time and within budget. Project managers, business analysts, user experience designers, and quality assurance engineers all collaborate with web developers.
  5. Monitoring Site Performance and Traffic:
    In addition to building new sites and applications, web developers can also serve as webmasters for their employer's internal websites. Webmasters work to update and maintain site functionality. This involves monitoring errors and metrics, resolving long load times, and making sure applications work properly

Small Computing Devices

What are small computing devices? What do they do?

Small computing devices also known as mobile computing are small portable computers, small wearable gadgets, and mobile devices which perform computing such as designing, developing, and building hardware and software systems or processing, structuring, and managing various kinds of information.
Small computing can be divided into at least three different categories of mobile computing:

  • Portable Computers
  • Mobile Phones
  • Wearable Computers
Portable computers like 2-in1 laptops allow people to perform similar tasks a PC (Personal Computer) can do without moving the whole computing device around and having trouble finding a place with a power supply for them to work on their tasks. This helps people save time and money as they do not need to travel back to the PC if they are going out and they certainly don’t need to buy a new PC if the work requires travelling to a distant workplace.

The mobile telephone, also called a mobile phone, is a portable device for connecting to a telecommunications network to transmit and receive voice, video, or other data. Generally, there are 2 types of mobile phones: feature phones, and smartphones. Smartphones are generally distinguished by touch-screen interfaces, fast internet access, a wide selection of installable applications, and a tall and wide but thin body. They've arguably come to equal or surpassing levels as desktop and laptop computers as today's dominant computing tools. present-day feature phones feature a phone and dialling pad that can be opened with the flip of a hand. They generally use physical buttons rather than touch screens. Some advanced feature phones today (Typically using KaiOS) have limited access to the Internet and come preloaded with Facebook, WhatsApp, and sometimes YouTube.

Wearable technology, also known as "wearables," is a category of electronic devices that can be worn as accessories, embedded in clothing, implanted in the user's body, or even tattooed on the skin. The devices are hands-free gadgets with practical uses, powered by microprocessors and enhanced with the ability to send and receive data via the Internet. The rapid adoption of such devices has placed wearable technology at the forefront of the Internet of things (IoT). Wearable devices such as smartwatches (Apple Watch, Samsung Watch, Fitbit) are very popular today. These wearables can receive calls or messages from the wearer’s smartphones. They also feature a wide array of sensors such as ECG (electrocardiogram), temperature sensors, altimeter, gyroscope, and accelerometers which can be useful in sporting activities.

Another small computing device that is widely used is Raspberry Pi. It is a single-board computer (SBC) developed by the Raspberry Pi Foundation to promote basic learning of computer science in schools in developing countries. Raspberry Pi isn’t the only widely used Single-Board Computer, other manufacturers offer many different single-board computers with competitive prices and even better hardware specifications, for example, the NVIDIA Jetson Nano, ASUS Tinker Board S R2.0, or the Odroid N2 Plus

What is the impact?

Rapid advancements in small computing and the widespread adoption of mobile devices will likely impact the way we perform many aspects of our daily lives. For example, in education, these events will likely have a significant impact on how we deliver education content, how we interact with our students and with each other, and even how our students learn. In business, today's workforce has become mobile. Mobile devices also allow companies to expand their workforce into a global network. Mobile devices allow colleagues to collaborate from anywhere in the world, removing any barriers to clear communication.

Smart mobile devices along with internet connections have made it possible to organise remote meetings, online meetings, video calls, and more. Mobile users can join team meetings and conference calls and contribute to the discussion as if they were physically present. This can help eliminate any delays in work and will also help meet deadlines, thus improving productivity in the workplace.

Such improved flexibility can help motivate employees to work harder and collaborate well with their colleagues.

Small computing has also made significant advances in the field of medicine, particularly in the 21st century. One good example is Capsule Endoscopy. It’s a procedure that allows the doctor to look inside the gastrointestinal tract (GI) using a small, wireless camera inside a small, safe, and disposable capsule. The capsule can be easily swallowed, and the camera will start taking a lot of pictures along its journey, the doctor can then diagnose GI tract conditions based on the pictures taken. This procedure is often used to take pictures of the small intestine. After 8 hours the camera capsule passes along with the feces.

In another corner of the small computing scene, the Raspberry Pi has helped kick-start a renewed interest in coding for young people through the development of online resources, coding clubs, programs, and competitions, which have reached millions of people from more than 100 countries. It runs two large networks of after-school clubs: CoderDojo and Code Club and produces a range of online resources to support self-directed learning around the world.

Machine Learning

What is machine learning?

Machine learning is a type of AI that utilises algorithms to mimic the way that humans learn with its accuracy improving the more it learns, automating many time consuming tasks As impressive as it is to have large processes automated, the machine learning model must first be trained in order to ensure that the model is working as intended. In order to improve its accuracy, the AI looks at previous data and makes decisions based on that data. This training can come in different forms: unsupervised training and supervised training. Unsupervised training requiring the model to seek out patterns and make its own conclusions from such data whereas supervised learning has humans label data per the criteria. Testing data will then be used to evaluate the model’s accuracy with the training and testing data coming from the data set often being split 80% training data and 20% testing data. If the outcome is not satisfactory, the model is then fed more data to better train it to become accurate. Of course, how well the model performs relies heavily on the testing data fed to it, the better the quality of the training data the better the accuracy of the machine learning model especially early on when it has less data to work with. Large companies such as Google, Amazon and Facebook utilise this to assist in understanding the customers’ behaviour.

An example of an implementation of machine learning would be recommendation systems which look at trends of other users’ activity which then recommends products to the user. It has become an integral part of many businesses, owing much success to machine learning. Speech recognition being another example is used everyday in mobile phones and other virtual assistants such as Amazon Alexa and Google Home. Through their speech recognition software, they are able to follow commands given completely hands free for the users’ convenience. Similarly, handwriting or image recognition looks at it’s recorded images and tries to match what it knows to what it sees. For example, when provided an image of a handwritten 8 it will look at its previous experiences for similarities to what it knows and then come to the conclusion that what it is seeing is an 8. Another application of this is photo tagging in social media which can detect a face and allow the user to tag their friends, or in self-driving cars such as Tesla’s cars which can identify traffic lights and signs in addition to pedestrians. There too are applications of chatbots being utilised in place of human customer support agents. The bots can answer FAQs and can provide personalised advice to a degree.

With the rapid increase in quantity of data machine learning became a feasible tool to be taken advantage of. As mentioned with data training, data is required for machine learning, however in order for effective training to be performed a substantial amount of data is required. With advancements in technology computing power has increased allowing for such large-scale solutions to exist. Today’s powerful GPUs have thousands of cores allowing for the calculations required for such immense amounts of data. Taking the cost of computing decreasing into consideration, organisations can utilise computing resources efficiently and cost effectively train complex machine learning algorithms with cloud computing being a major factor. With access to the cloud, businesses are able to access their resources on demand allowing for the workload of the computations to be shared.
